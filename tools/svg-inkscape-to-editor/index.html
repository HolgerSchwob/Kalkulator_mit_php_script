Inkscape → Editor (Postprozessor)

SVG aus Inkscape laden (mit $$F1$$/$$S1$$-Markern, Ebenen, Codierfarben) → Konvertieren → Editor-taugliche SVG mit IDs und Gruppen. Anschließend mit Musterdaten testen.

SVG ablegen oder klicken

SVG-Datei wählen (aus Inkscape exportiert, mit $$F1$$ etc., Ebenen, Codierfarben 80/70/80 und 40/30/40).

Workflow: Von der Erstellung bis zur Production-SVG
Stufe 1 – Postprozessor (dieses Tool) • Inkscape: Template mit $$F1$$/$$S1$$-Markern, Ebenen und Codierfarben erstellen. • In Inkscape nur als „Inkscape SVG“ oder „Plain SVG“ speichern – nicht als „Optimiertes SVG“. Sonst gehen Ebenen und Metadaten verloren und der Postprozessor kann keine Gruppen zuweisen. • Hier: SVG laden → Konvertieren → Ergebnis herunterladen (z. B. …-editor.svg). Optional mit Musterdaten testen. Stufe 2 – Productioner (Schriften + Optimierung) • Die editor-SVG im Tool „SVG Productioner“ öffnen. • Schriften werden eingebettet und gleichzeitig unnötige Metadaten (inkscape:, sodipodi: etc.) entfernt. • Als Production-SVG speichern (druckfertig). Danach • Optional: SVG Template Preflight zur Prüfung (Boxen, Text-Platzhalter, Farben). • Production-SVG im Dashboard hochladen (Einstellungen → Templates). Wichtig: Optimierung/Bereinigung immer nach dem Postprozessor. Nie optimierte SVG aus Inkscape direkt in den Postprozessor laden.

Spickzettel Inkscape (Marker, Ebenen, Farben)

Texte: In Inkscape am Textanfang tippen: $$F1$$ = Titel, $$F2$$ = Untertitel, $$F3$$ = Name, $$F4$$ = Matrikelnummer, $$F5$$ = Jahr, $$F6$$ = Thema, $$F7$$ = Thema mehrzeilig, $$F8$$–$$F10$$ = Untertitel Zeile 1–3. Buchrücken: $$S1$$ = Titel, $$S2$$ = Name, $$S3$$ = Thema.

Ebenen: Ebenenname → Gruppe: „Coverfront“/„Vorderseite“/„Front“/„U1“ → tpl-group-u1; „Coverback“/„Rückseite“/„U4“ → tpl-group-u4; „Spine“/„Buchrücken“/„Rücken“ → tpl-group-spine.

Farben: RGB 80/70/80 (#504650) = color1, RGB 40/30/40 (#281e28) = color2. Diese Füllung in Inkscape verwenden, dann werden sie automatisch zugeordnet.

Skalierungsboxen (tpl-*_bbox)
Für automatische Schriftanpassung im Editor können in Inkscape unsichtbare Rechtecke mit ID tpl-TEXTID_bbox angelegt werden (z. B. tpl-title_bbox, tpl-topic_bbox). Der Postprozessor erzeugt diese nicht automatisch – bei Bedarf in Inkscape manuell hinzufügen.
