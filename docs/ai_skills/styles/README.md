# Style-Skills: reine Optik pro Archetyp

**Zweck:** Kleine, fokussierte Anweisungen **nur für den visuellen Stil** eines Template-Typs. Keine Duplikation von Maßen, IDs, BBox, Falz — das steht ausschließlich in **`../SKILL_01_TECHNICAL.md`** und **`../../SSOT_SVG_COVER_TEMPLATES.md`**.

## Schichten (Reihenfolge für Agent & KI)

1. **`docs/SSOT_SVG_COVER_TEMPLATES.md`** — IDs, Gruppen, Webshop-Konventionen  
2. **`docs/ai_skills/SKILL_01_TECHNICAL.md`** — Technik (Pflicht)  
3. **`docs/ai_skills/SKILL_02_DESIGN.md`** — globale Typo-Bänder, Paletten-Übersicht, Art Direction  
4. **`SKILL_STYLE_<ID>_<KURZNAME>.md`** (dieser Ordner) — **nur** Layout/Optik dieses Stils  

Externe KI für „nur Gestaltung“ kann **(3)+(4)** erhalten; Technik bleibt **(1)+(2)** im Projekt.

## Namenskonvention

`SKILL_STYLE_<Archetyp-ID>_<KURZNAME>.md`

Beispiel: `SKILL_STYLE_L05_SWISS_EDITORIAL.md`

## Neues Style-Skill anlegen

Vorlage: **`SKILL_STYLE_SCAFFOLD.md`** kopieren, Datei nach Namenskonvention speichern und eine Zeile in der Tabelle **„Layout-Archetypen — Index“** in [`SKILL_02_DESIGN.md`](../SKILL_02_DESIGN.md) **Abschnitt 2** ergänzen (ID, Kurzbeschreibung, Link auf das neue `SKILL_STYLE_*.md`).
